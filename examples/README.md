# Sample Charts

These SVG charts are generated by GitHub Star Tracker. By default they adapt to dark and light mode via `@media (prefers-color-scheme: dark)`; you can force a palette with `chart-theme: light|dark`.

Toggle your system or browser theme to see the effect.

Chart appearance is configurable — the curve style, smoothing, point markers, animation, milestone lines (built-in or custom), the Y-axis floor, the time window, and a moving-average trend line. See **[Star Trend Charts](../docs/wiki/Star-Trend-Charts.md)** for the full list.

---

## Curve Styles

`chart-curve` picks how the line is drawn between points. The examples below use the same stepped data so the differences are visible. `monotone` is the default: it stays smooth without ever dipping below a value, which suits star counts since they only go up.

| `monotone` (default) | `catmull-rom` |
|---|---|
| ![Monotone](curve-monotone.svg) | ![Catmull-Rom](curve-catmull-rom.svg) |

| `cubic-bezier` | `rounded-step` |
|---|---|
| ![Cubic Bezier](curve-cubic-bezier.svg) | ![Rounded Step](curve-rounded-step.svg) |

Notice how `catmull-rom` overshoots at the foot of each step (the line dips below the previous value), while `monotone`, `cubic-bezier` and `rounded-step` keep the plateaus flat.

Email charts (rendered via QuickChart) respect `chart-curve` too, with one caveat: `monotone` is reproduced exactly, `rounded-step` falls back to `monotone`, and `catmull-rom` and `cubic-bezier` both render as a tensioned spline.
